## Working Rules

1. keep the public API smaller and simpler than ESP-IDF
2. preserve one common API across supported targets
3. keep examples, docs, and shipped behavior aligned
4. prefer small additive changes over broad redesigns
5. validate affected areas on all supported targets when required

## Documentation Rules

- keep user docs task-first
- keep module docs reference-first
- avoid duplicating setup, build, and flash instructions across many pages
- keep contributor/process material out of the main user navigation unless necessary
