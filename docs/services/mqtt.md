# MQTT Client

Publish/subscribe messaging client for connecting to an MQTT broker over WiFi.



## Target Support

| Target | Supported |
|--------|-----------|
| ESP32 | yes |
| ESP32-C3 | yes |
| ESP32-S3 | yes |

## Types

### `blusys_mqtt_qos_t`

```c
typedef enum {
    BLUSYS_MQTT_QOS_0 = 0,
    BLUSYS_MQTT_QOS_1 = 1,
    BLUSYS_MQTT_QOS_2 = 2,
} blusys_mqtt_qos_t;
```

MQTT Quality of Service level. QoS 0 is fire-and-forget; QoS 1 guarantees at-least-once delivery; QoS 2 guarantees exactly-once delivery.

### `blusys_mqtt_message_cb_t`

```c
typedef void (*blusys_mqtt_message_cb_t)(const char    *topic,
                                         const uint8_t *payload,
                                         size_t         payload_len,
                                         void          *user_ctx);
```

Callback invoked from the MQTT client task when a message arrives on a subscribed topic. `topic` is null-terminated. `payload` is raw bytes and is **not** null-terminated; use `payload_len` to determine length. The callback must not block for extended periods.

### `blusys_mqtt_config_t`

```c
typedef struct {
    const char               *broker_url;     /* required; e.g. "mqtt://broker.example.com:1883" */
    const char               *username;        /* NULL = no authentication */
    const char               *password;        /* NULL = no authentication */
    const char               *client_id;       /* NULL = auto-generated by ESP-IDF */
    const char               *server_cert_pem; /* PEM CA cert for MQTTS; NULL skips verification */
    int                       timeout_ms;      /* connect timeout; BLUSYS_TIMEOUT_FOREVER to block */
    blusys_mqtt_message_cb_t  message_cb;      /* called on incoming messages; NULL = no subscription handling */
    void                     *user_ctx;        /* passed through to message_cb */
} blusys_mqtt_config_t;
```

`server_cert_pem` must remain valid for the lifetime of the handle (the pointer is not copied).

## Functions

### `blusys_mqtt_open`

```c
blusys_err_t blusys_mqtt_open(const blusys_mqtt_config_t *config, blusys_mqtt_t **out_handle);
```

Allocates and initialises an MQTT client. Does not connect to the broker.

**Parameters:**
- `config` — broker URL, credentials, TLS cert, timeout, and message callback (required)
- `out_handle` — output handle

**Returns:** `BLUSYS_OK` on success, `BLUSYS_ERR_INVALID_ARG` if config or broker_url is NULL or timeout_ms is invalid, `BLUSYS_ERR_NO_MEM` on allocation failure, `BLUSYS_ERR_NOT_SUPPORTED` on targets without WiFi.

---

### `blusys_mqtt_close`

```c
blusys_err_t blusys_mqtt_close(blusys_mqtt_t *handle);
```

Disconnects from the broker if still connected and frees the handle. After this call the handle is invalid.

---

### `blusys_mqtt_connect`

```c
blusys_err_t blusys_mqtt_connect(blusys_mqtt_t *handle);
```

Connects to the broker. Blocks until CONNACK is received, the `timeout_ms` set at open time expires, or a network error occurs.

**Returns:**
- `BLUSYS_OK` — broker accepted the connection
- `BLUSYS_ERR_INVALID_ARG` — NULL handle
- `BLUSYS_ERR_TIMEOUT` — broker did not respond within the timeout
- `BLUSYS_ERR_IO` — network-level failure (DNS, TCP, TLS handshake, or broker refusal)

---

### `blusys_mqtt_disconnect`

```c
blusys_err_t blusys_mqtt_disconnect(blusys_mqtt_t *handle);
```

Disconnects from the broker. Returns `BLUSYS_OK` immediately if not connected.

---

### `blusys_mqtt_publish`

```c
blusys_err_t blusys_mqtt_publish(blusys_mqtt_t *handle,
                                  const char    *topic,
                                  const uint8_t *payload,
                                  size_t         payload_len,
                                  blusys_mqtt_qos_t qos);
```

Publishes a message. Fire-and-forget: QoS acknowledgments are handled by the ESP-IDF MQTT stack internally and are not surfaced to the caller.

**Returns:**
- `BLUSYS_OK` — message handed to the MQTT stack
- `BLUSYS_ERR_INVALID_ARG` — NULL handle or topic
- `BLUSYS_ERR_INVALID_STATE` — not connected
- `BLUSYS_ERR_IO` — internal MQTT client error

---

### `blusys_mqtt_subscribe`

```c
blusys_err_t blusys_mqtt_subscribe(blusys_mqtt_t *handle,
                                    const char    *topic,
                                    blusys_mqtt_qos_t qos);
```

Subscribes to a topic filter. Incoming messages are delivered via the `message_cb` set at open time.

**Returns:**
- `BLUSYS_OK` — subscription request sent
- `BLUSYS_ERR_INVALID_ARG` — NULL handle or topic
- `BLUSYS_ERR_INVALID_STATE` — not connected
- `BLUSYS_ERR_IO` — internal MQTT client error

---

### `blusys_mqtt_unsubscribe`

```c
blusys_err_t blusys_mqtt_unsubscribe(blusys_mqtt_t *handle, const char *topic);
```

Unsubscribes from a topic filter.

**Returns:** same as `blusys_mqtt_subscribe`.

---

### `blusys_mqtt_is_connected`

```c
bool blusys_mqtt_is_connected(blusys_mqtt_t *handle);
```

Returns `true` if the client currently has an active connection to the broker. Safe to call without holding the internal lock.

## Lifecycle

```
blusys_mqtt_open()
    └── blusys_mqtt_connect()
            └── blusys_mqtt_subscribe() / blusys_mqtt_publish()
                    [repeat as needed]
        blusys_mqtt_disconnect()
blusys_mqtt_close()
```

The handle may be reconnected after a `disconnect()` by calling `connect()` again.

## Thread Safety

`publish()`, `subscribe()`, and `unsubscribe()` serialise concurrent callers via an internal lock. `open()` and `close()` must not be called concurrently with any other function on the same handle.

## Limitations

- The WiFi connection must be established before calling `connect()`.
- `publish()` is fire-and-forget — there is no acknowledgment confirmation at the blusys API level.
- The `message_cb` is called from the MQTT client task; it must return promptly and must not call `connect()`, `disconnect()`, or `close()` on the same handle.
- MQTTS certificate verification requires a valid PEM cert in `config.server_cert_pem`; passing `NULL` skips verification.
