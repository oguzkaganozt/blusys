# Examples

Examples are organized into three categories:

## quickstart/

Canonical product starters that demonstrate the recommended `blusys::app` path. These are the first examples new users should look at.

- **connected_device** — interactive connected product using the app runtime, display, and connectivity bundles
- **connected_headless** — headless connected product using the same reducer model without a display
- **framework_app_basic** — minimal interactive app quickstart
- **framework_ui_basic** — interactive app with stock widgets and bindings
- **framework_device_basic** — interactive app targeting a real device profile

## reference/

Scoped capability demos. Each example demonstrates one specific module or feature and serves as a standalone reference for that capability.

## validation/

Internal stress tests, concurrency tests, and peripheral smoke tests. These are not part of the public learning surface — they exist to validate the platform on CI and real hardware.
